## 代码分析方向
```md
规约扫描 规约是历史错误经验的总结，衡量代码与集团规约扫描的符合程度
重复度 重复代码越多，代码越难维护，修改代码所带来的风险越高
复杂度 单元（方法或类）复杂度越高，代码越难维护，出现故障的可能性越高
面向对象 面向对象设计的好坏，是软件是否容易维护的重要影响因素
```
