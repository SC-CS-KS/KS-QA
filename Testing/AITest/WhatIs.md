# AI Test

```md
数据准备，结果校验都交给机器自动完成。
```
## 六个层次
```md
完全没有自动 驾驶辅助——AI 可以查看到页面，帮助你写出断言
部分自动化 在这种模式下，AI逐渐具备了贯穿上下文的能力
条件自动化 在这种模式下，AI逐渐具备了自我学习的能力
高度自动化 核心改变就是从人工”驱动“发展为”AI“驱动
完全自动化 发展到这个阶段，测试这个岗位需要重新被定义。
```
## 运用场景
```md
探索AI和ML（机器学习）技术能如何被运用到测试场景

Unit Testing
	单元测试对于确保构建是稳定和可测试的非常重要。
	使用像RPA这样的AI支持的单元测试工具，开发人员可以减少脆弱的测试用例和单元测试的维护。

API Testing
	帮助测试人员对手工UI测试自动转换为API测试
	自动生成和维护API测试

UI Testing
	如何把手工测试用例转换为自动化测试用例，AI技术在此场景下目前大多被运用在结果识别以及多场景的适配测试领域，
    从而降低对UI自动化的维护和运行成本。
```
